## 为什么我要搭建个人博客？

作为一名独立游戏开发者，我一直想拥有一个属于自己的技术博客。

不是寄居在CSDN或掘金上的一亩三分地——那些平台虽然方便，但你的内容被算法控制，页面充斥着广告，而且你永远不知道哪天平台会改变规则。

我想要的是一个**完全属于自己的空间**：
- 记录游戏开发日志
- 沉淀技术学习笔记
- 展示个人作品和项目
- 建立自己的个人品牌

经过一番折腾，我成功搭建了现在这个博客。整个过程从零到上线大约花了一个下午的时间。这篇文章完整记录了我的搭建过程，希望能帮助同样想要拥有个人博客的你。

---

## 技术选型：为什么是这套方案？

市面上搭建博客的方案很多，我最终选择了 **Hugo + GitHub Pages + Stack主题** 的组合。让我解释为什么：

### Hugo：静态网站生成器

| 对比项 | Hugo | WordPress | Hexo |
|--------|------|-----------|------|
| 构建速度 | 极快（毫秒级） | 慢（需要服务器渲染） | 较快（秒级） |
| 部署方式 | 纯静态文件 | 需要PHP+数据库 | 纯静态文件 |
| 技术门槛 | 中等 | 低 | 中等 |
| 运行成本 | 免费 | 需要买服务器 | 免费 |
| 安全性 | 极高（无后端） | 较低（常被攻击） | 极高 |

Hugo用Go语言编写，构建速度是同类工具中最快的。我的博客构建时间只需要几十毫秒——这意味着每次保存文件，页面几乎瞬间刷新。

### GitHub Pages：免费托管

GitHub Pages是GitHub提供的静态网站托管服务，完全免费，自带HTTPS，而且对开发者非常友好。你的博客代码就是一个Git仓库，天然具备版本控制能力。

### Stack主题：现代个人品牌风

Stack是Hugo生态中最受欢迎的博客主题之一。它有侧边栏布局、暗色模式切换、标签云、文章归档等功能，非常适合打造个人品牌页面。

## 准备工作

在正式开始之前，我们需要准备三样东西：Git、GitHub账号和Hugo。

### 1. 什么是Git？为什么需要它？

Git是一个版本控制工具，简单来说就是帮你管理代码历史的工具。你可以把它想象成一个**超级存档系统**——每次你做了修改并"提交"（commit），Git就帮你保存一个存档点。如果改坏了，可以随时回退。

在搭建博客的过程中，Git的作用是：
- 管理博客的所有文件
- 将文件推送到GitHub
- 配合GitHub Actions实现自动部署

**安装Git：**

去 [git-scm.com](https://git-scm.com/) 下载Windows版本，安装时一路默认即可。

安装完成后，打开PowerShell验证：

`powershell
git --version
`

看到类似 `git version 2.44.0` 的输出就说明安装成功了。

**首次使用需要配置身份信息：**

`powershell
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
`

这个信息会显示在你的每次提交记录中。

### 2. 什么是GitHub？

GitHub是全球最大的代码托管平台，你可以把它理解为"程序员的云盘"。我们用它来存放博客的源代码，并利用它的GitHub Pages功能来托管博客网站。

**注册GitHub：**

去 [github.com](https://github.com/) 注册账号。

> **重要提示：** 你的用户名会成为博客地址的一部分。比如用户名是 `muwustudio`，博客地址就是 `muwustudio.github.io`。建议选择一个简洁、有辨识度、和你个人品牌相关的用户名。

### 3. 安装Hugo

打开PowerShell，执行：

`powershell
winget install Hugo.Hugo.Extended
`

安装完成后重启PowerShell，验证：

`powershell
hugo version
`

> **注意：** 必须安装 **Extended** 版本（命令中已经指定了）。Stack主题使用SCSS来编写样式，只有Extended版本才支持SCSS编译。如果安装了普通版本，主题会报错。

## Step 1：创建Hugo站点

打开PowerShell，执行以下命令：

`powershell
# 创建新站点（把my-blog替换成你想要的文件夹名）
hugo new site my-blog

# 进入项目目录
cd my-blog

# 初始化Git仓库
git init
`

执行完成后，你会看到Hugo自动生成了一个项目结构。其中我们最常打交道的是：

`
my-blog/
├── content/        ← 你的文章放在这里
├── static/         ← 静态文件（图片、CSS等）
├── themes/         ← 主题放在这里
└── hugo.toml       ← 博客的配置文件
`

---

## Step 2：安装Stack主题

### 方式一：Git克隆（推荐）

`powershell
git clone https://github.com/CaiJimmy/hugo-theme-stack.git themes/stack
`

### 方式二：手动下载（Git失败时的备选方案）

如果你在国内，GitHub访问可能不稳定。这时可以手动下载：

1. 浏览器打开 `https://github.com/CaiJimmy/hugo-theme-stack`
2. 点击绿色的 **Code** 按钮
3. 点击 **Download ZIP**
4. 下载后解压
5. 将解压出的文件夹**重命名为 `stack`**
6. 放到项目的 `themes/` 目录下

> **特别注意：** 解压后文件夹名通常是 `hugo-theme-stack-master`，你必须改成 `stack`。否则Hugo找不到主题。最终确认 `themes/stack/theme.toml` 这个文件存在就对了。

## Step 3：配置博客

Hugo使用一个配置文件来控制博客的所有设置。用记事本或代码编辑器打开项目根目录的 `hugo.toml`（如果不存在就新建一个）。

将内容替换为：

`toml
baseURL = 'https://你的用户名.github.io/'
languageCode = 'zh-cn'
title = '你的博客名称'
theme = 'stack'
paginate = 10
hasCJKLanguage = true

[params]
mainSections = ['posts']
rssFullContent = true

[params.footer]
since = 2026

[params.sidebar]
emoji = '🏠'
subtitle = '你的一句话介绍'
avatar = ''

[params.article]
math = false
toc = true
readingTime = true

[params.colorScheme]
toggle = true
default = 'auto'

[params.comments]
enabled = false
`

**需要修改的地方：**
- `baseURL`：替换成你的GitHub用户名
- `title`：你的博客名称
- `emoji`：选一个代表你的emoji
- `subtitle`：一句话介绍自己

> **编码注意事项：** 保存文件时必须使用 **UTF-8** 编码。如果使用Windows记事本，保存时在"编码"下拉框中选择 `UTF-8`（不是"带BOM的UTF-8"）。推荐使用VS Code或Cursor等现代编辑器，它们默认就是UTF-8编码。

## Step 4：创建菜单页面

为了让博客有完整的导航，我们需要创建几个基础页面。

### 归档页面（按时间线展示所有文章）

`powershell
mkdir -p content/page/archives
`

创建 `content/page/archives/index.md`，内容为：

`markdown
---
title: "归档"
date: 2026-03-25
layout: "archives"
slug: "archives"
---
`

### 关于页面

`powershell
mkdir -p content/page/about
`

创建 `content/page/about/index.md`，写上你的个人介绍。

### 配置菜单

在 `hugo.toml` 中添加菜单配置：

`toml
[menu]
[[menu.main]]
identifier = 'home'
name = '首页'
url = '/'
weight = -100
[menu.main.params]
icon = 'home'

[[menu.main]]
identifier = 'archives'
name = '归档'
url = '/archives/'
weight = -90
[menu.main.params]
icon = 'archives'

[[menu.main]]
identifier = 'about'
name = '关于'
url = '/about/'
weight = -50
[menu.main.params]
icon = 'user'
`

> **注意：** 菜单图标名称必须与主题 `themes/stack/assets/icons/` 目录下的SVG文件名一致。可以用 `dir themes/stack/assets/icons` 查看所有可用图标。

## Step 5：撰写第一篇文章

`powershell
hugo new posts/hello-world.md
`

打开 `content/posts/hello-world.md`，你会看到Hugo自动生成了一些内容：

`markdown
---
title: "Hello World"
date: 2026-03-25T15:00:00+08:00
draft: true
---
`

修改成你想要的内容：

`markdown
---
title: "你好，世界！这是我的第一篇博客"
description: "介绍我自己和这个博客"
date: 2026-03-25
tags:
  - 自我介绍
categories:
  - 随笔
draft: false
---

## 关于我

写一段自我介绍...

## 这个博客会记录什么

写一下你的博客规划...
`

> **⚠️ 最关键的一步：** 必须把 `draft: true` 改为 `draft: false`！Hugo默认将新文章设为草稿状态，草稿文章在正式构建时不会显示。这是新手最容易踩的坑。

### 添加封面图（可选但推荐）

在front matter中添加 `image` 字段可以为文章设置封面图：

`markdown
---
title: "文章标题"
image: https://images.unsplash.com/photo-xxxxx?w=1200
---
`

推荐去 [unsplash.com](https://unsplash.com/) 寻找免费高质量图片。封面图会让你的博客首页更加美观。

## Step 6：本地预览

`powershell
hugo server
`

打开浏览器，访问 `http://localhost:1313`。

你应该能看到：
- 左侧边栏显示你的博客名称和emoji
- 主区域显示你的文章列表
- 右上角有亮色/暗色模式切换按钮

每次你修改文件并保存，Hugo会自动重新构建，浏览器会实时刷新。这个热重载功能在写文章时非常方便。

确认效果满意后，按 `Ctrl+C` 停止本地服务器。

## Step 7：部署到GitHub Pages

### 7.1 创建GitHub仓库

1. 登录GitHub
2. 点击右上角 **+** → **New repository**
3. 仓库名填：`你的用户名.github.io`
4. 选择 **Public**
5. **不要勾选**任何初始化选项

> **仓库名必须严格等于 `用户名.github.io`，这是GitHub Pages的要求。**

### 7.2 配置SSH连接（推荐）

SSH比HTTPS更稳定，尤其是在国内网络环境下。

**生成SSH密钥：**

`powershell
ssh-keygen -t ed25519 -C "你的邮箱@example.com"
`

一路按回车（不需要设置密码）。

**复制公钥：**

`powershell
cat ~/.ssh/id_ed25519.pub
`

复制输出的全部内容。

**添加到GitHub：**

1. GitHub → 右上角头像 → **Settings**
2. 左侧栏 → **SSH and GPG keys**
3. 点击 **New SSH key**
4. Title随便填，Key粘贴刚才复制的内容
5. 保存

**测试连接：**

`powershell
ssh -T git@github.com
`

看到 `Hi xxx! You've successfully authenticated` 就成功了。

> **国内网络问题解决方案：** 如果SSH连接提示 `Connection refused`，这是因为22端口被运营商屏蔽了。解决方法是让SSH走443端口。
>
> 创建或编辑文件 `~/.ssh/config`，写入以下内容：
>
> `
> Host github.com
>     Hostname ssh.github.com
>     Port 443
>     User git
> `
>
> 保存后重新测试 `ssh -T git@github.com`，应该就能通了。

### 7.3 配置GitHub Actions自动部署

在项目中创建部署配置文件：

`powershell
mkdir -p .github/workflows
`

创建 `.github/workflows/hugo.yml`，内容为：

`yaml
name: Deploy Hugo Blog

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: true
          fetch-depth: 0

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: 'latest'
          extended: true
    
      - name: Build
        run: hugo --minify
    
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
`

这个配置的作用是：每次你push代码到main分支，GitHub会自动运行Hugo构建网站，并将生成的静态文件部署到gh-pages分支。

### 7.4 推送代码

`powershell
git add .
git commit -m "初始化博客"
git remote add origin git@github.com:你的用户名/你的用户名.github.io.git
git push -u origin main
`

### 7.5 配置GitHub Pages

1. 进入你的GitHub仓库
2. 点击 **Settings** → 左侧栏 **Pages**
3. Source选择 **Deploy from a branch**
4. Branch选择 **gh-pages**，文件夹选 **/ (root)**
5. 点击 **Save**

> **⚠️ 重要：** 首次部署前，需要在仓库的 Settings → Actions → General → Workflow permissions 中选择 **Read and write permissions** 并保存。否则GitHub Actions没有权限创建gh-pages分支，部署会失败。

等待2-3分钟，访问 `https://你的用户名.github.io`，你的博客就正式上线了！

## 以后的日常工作流

博客搭建完成后，以后发布文章的流程非常简单：

`powershell
# 1. 创建新文章
hugo new posts/文章名.md

# 2. 编辑文章内容（记得把draft改为false）

# 3. 本地预览
hugo server

# 4. 满意后发布
git add .
git commit -m "新文章：文章标题"
git push
`

推送后GitHub Actions会自动构建部署，大约2-3分钟后文章就上线了。整个过程只需要四步，非常高效。

## 关于国内访问

需要坦诚地说：GitHub Pages在国内的访问速度不稳定，有时甚至无法打开。

**我的解决方案是双平台运营：**

- **GitHub Pages**：作为主站，面试时展示给面试官看（技术圈的人基本都能访问）
- **CSDN / 掘金**：同步发布同样的文章，国内用户可以流畅阅读

每篇文章用Markdown写一次，然后分别发布到两个平台。一份内容，两个渠道。

如果未来想要更好的国内访问体验，可以考虑购买域名并配置CDN加速，但这属于进阶操作，不在本文的范围内。

我给自己定的目标是：**每完成一个项目模块，就写一篇开发日志**。不追求完美，不追求阅读量，只追求真实和持续。
