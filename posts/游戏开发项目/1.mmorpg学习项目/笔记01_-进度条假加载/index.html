笔记01_进度条假加载

[Header] 和 [Tooltip]

这两个是 Inspector 可视化特性，写在变量上方，不影响任何逻辑。

[Header("UI 组件")]          // 在Inspector中显示分类标题
[Tooltip("这是进度条")]       // 鼠标悬停时显示说明文字
public Slider progressSlider;

🗣️ 口诀：Header 是分组标题，Tooltip 是鼠标提示，纯粹美化 Inspector 用。

1. ⚙️ 协程 IEnumerator

📖 简洁解释

普通函数：一帧内从头跑到尾，中途无法暂停。 协程：可以在中途暂停，等条件满足后再继续，不会卡住画面。

💻 最简代码示例

// ==============================
// 普通函数 —— 进度条根本看不到变化，画面直接卡到100%
// ==============================
void BadExample()
{
    for (int i = 0; i <= 100; i++)
    {
        progressSlider.value = i; // 画面来不及刷新，玩家看不到过程
    }
}

// ==============================
// 协程 —— 每帧更新一次，画面正常刷新
// ==============================
IEnumerator GoodExample()
{
    for (int i = 0; i <= 100; i++)
    {
        progressSlider.value = i;
        yield return null;             // ← 暂停，等下一帧再继续
    }
}

// ==============================
// 协程的启动方式（必须用StartCoroutine！）
// ==============================
void Start()
{
    StartCoroutine(GoodExample());     // ← 不能直接调用，必须这样启动
}

常用 yield 暂停方式：

yield return null;                     // 等待下一帧
yield return new WaitForSeconds(2f);   // 等待2秒

🗣️ 口诀：需要"等待"或"逐帧更新"，就用协程，用 yield return 控制暂停。

📌 常见使用场景

场景例子
逐帧更新画面进度条、血条动画、技能CD
等待一段时间3秒后关闭提示框、延迟跳转场景
等待某件事完成等待异步加载场景完毕
技能/动画序列先播放动画，再造成伤害

2. 🔢 Mathf.RoundToInt()

📖 简洁解释

Mathf 是 Unity 内置数学工具箱。 RoundToInt = Round（四舍五入）+ ToInt（转成整数）。 因为 Slider 的 value 是 float（小数），直接显示会出现 99.9998% 这种情况，用它转成整数显示更干净。

💻 最简代码示例

float value = 99.6f;

// ✗ 不处理直接显示
progressText.text = value + "%";            // 显示：99.6%（甚至 99.999847%）

// ✓ 四舍五入转整数后显示
int result = Mathf.RoundToInt(value);       // result = 100
progressText.text = result + "%";           // 显示：100%

🗣️ 口诀：Mathf 是数学工具箱，float 转 int 显示用 RoundToInt，四舍五入更美观。

📌 常见使用场景

场景例子
进度条百分比显示99.9f → 100%
伤害数字显示避免显示小数伤害值
经验值/金币计算百分比经验条

3. 🚪 SceneManager.LoadScene()

📖 简洁解释

Unity 把"跳转场景"的功能封装在 SceneManager（场景管理器）里， LoadScene = Load（加载）+ Scene（场景），见名知意。 使用前需要在代码顶部引入命名空间。

💻 最简代码示例

// 第一步：顶部引入命名空间（必须加！否则报错）
using UnityEngine.SceneManagement;

// 第二步：调用跳转
SceneManager.LoadScene("MainGame");    // 括号内填写目标场景的名字（字符串）

// ⚠️ 注意：场景必须先在 Build Settings 中添加才能跳转
// 菜单：File → Build Settings → 把场景拖进去

🔍 以后遇到类似不会的API，怎么找？

第一步：用中文描述需求
   └─ "Unity 怎么切换场景"

第二步：问 AI 或查 Unity 官方文档
   └─ docs.unity3d.com（搜索框直接搜 SceneManager）

第三步：看 API 名字，Unity 命名非常语义化，见名知意
   └─ SceneManager.LoadScene
      ↑场景管理器  ↑加载场景    ← 光看名字就能猜到意思

第四步：确认需要 using 哪个命名空间
   └─ 文档里会写，或者直接问 AI

🗣️ 口诀：跳转场景找 SceneManager，用之前加 using，场景名要在 Build Settings 里注册。

📌 常见使用场景

场景例子
加载完毕后跳转假加载结束 → 进入主城
点击按钮切换场景主菜单 → 开始游戏
角色死亡后重载重新加载当前场景
